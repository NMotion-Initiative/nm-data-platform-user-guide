Your first WBCD submission

Goal of this section: get one labeled episode out the door, end-to-end, in about ten minutes. The deep references are below if you need them, but you don't need them yet.

  1. Sign in. You land on the project picker.
  2. Create your team's project. Click + New project, name it something like wbcd-2026-team-xyz. You'll drop into the empty workspace.
  3. Pull in starter data. Sidebar → My Data → Imports tab. Pick a source (SFTP for contest-provided data, Hugging Face for public reference sets, file upload for your own HDF5). The job lands in the folder you've selected on the left.
  4. Open one episode. Switch to the Files tab. Double-click any episode row to jump into the editor.
  5. Mark a sub-goal segment. Scrub the timeline to the start of a phase (e.g. the moment the gripper closes). Press R to switch to the tag tool. Click the timeline at the start, then click again at the end. A coloured span appears.
  6. Name it. The right panel opens to Sub-goal segments. Fill in the Sub-goal name ("Grasp Cube") and the Language instruction ("Close the gripper around the cube and lift to 12cm").
  7. Save your edit. Top right of the editor → Save edit. Now you have a saved version of this episode with your sub-goal segment attached.
  8. Repeat for the rest of the phases. Keep adding segments and saving. Three to six segments per episode is typical.
  9. Export the LeRobot bundle. Back in My Data → Exports tab. Pick the LeRobot format tile, name your version, hit Queue export.
  10. Submit. When the export row finishes (status pill turns ready), download the file and upload it through the WBCD submission portal.
If you only read one more section

Read §08 · Editing. That's where the bulk of your time goes.

01

Catalog vs your project

The single mental model you need:

Catalog A shared library of robot-learning datasets — public LeRobot sets, vendor-shipped sets, and anything your workspace uploaded. Read-only browsing. Lives in the Dataset Explorer sidebar item.
Your project Your own bookshelf. Only the data you've explicitly pulled in. Where you edit, label, and export from. Lives in the My Data sidebar item.

Browsing the catalog doesn't put anything in your project. You add data with the Add to project action — see §06.

An episode is one recorded trial (frames, cameras, robot states). A dataset is a collection of episodes from one source. Folders are how you organise things inside your project. That's the whole vocabulary.

Day to day
02

Sign in & pick a project

After sign-in you land on the project picker. You can't enter the app until you pick or create a project — everything that follows is scoped to one.

  • Pinned — projects you've touched in the last week. Click a card to open it as a new tab.
  • + New project — for a fresh competition project, name it then drop into the empty workspace.
  • Recent — anything else you've opened in the last 30 days, including archived.
03

Workspace shell

Projects open as tabs across the top. Work like browser tabs. Click to switch, ✕ to close (the project isn't deleted — it just moves to Recent projects ⌄). The topbar shows the breadcrumb of where you are, plus global tools: ⌘K search, theme toggle, and notifications.

1 Dataset Explorer — the shared dataset catalog.
2 My Data — your project's file manager.
3 Editing — annotation editor. Badge = job count.
4 Activity — events across your projects.
5 Project members — invite, set roles.
6 Settings + Help — utility row above your avatar.
05

Browse the catalog

Dataset Explorer is the shared library. Datasets are grouped by vendor. Search by title, provider, or tag.

  • Click a card to open the dataset. Its episodes appear in the right inspector.
  • Click any episode row to scrub through it. Double-click to open in the editor.
  • Use the tag icon on a card to pin a dataset to the top of the catalog.
06

Add data to your project

Hover over a dataset card and tick the checkbox. To pick specific episodes, open a dataset and tick the checkboxes on individual episode rows. You can mix both kinds of selection.

When you have one or more items selected, a toolbar appears at the bottom of the screen. Click ⌄ Add to project, pick a folder, hit Add. The items show up in My Data immediately.

Items already in the destination folder are skipped, not duplicated. You'll see a count of skipped duplicates in the success toast.

07

My Data — files, imports, exports, history

My Data is everything in your project plus everything flowing in and out of it. Folder tree on the left, a four-tab strip on the right: Files (default) · Imports · Exports · History.

Files tab

Contents of the selected folder, in three sections: sub-folders, datasets, and individual episodes. Double-click any episode row to open the editor. Single-click to select for bulk actions (move, flag, favorite, remove).

+ New folder adds a folder; right-click any folder row to rename / move / delete. Quick filters below the tree are saved views — Flagged, Favorited, Recently added — they show matching items wherever they live.

Imports tab

Pick a source at the top, watch jobs run at the bottom. Imports land in whichever folder is selected in the tree.

SourceFor
Upload fileDrag-and-drop HDF5 / tar / parquet from your laptop. Up to 12 GB per file.
S3 prefixs3://bucket/prefix/. Your S3 bucket needs to grant read access to NM's importer — see the S3 setup guide for the exact policy text.
Hugging FacePull a public LeRobot dataset by repo id.
SFTP serverFor contest-provided data. Uses SSH keys (we'll give you our public key during setup).

Every import passes through five stages: download → extract → index file → preview thumbnails → make searchable. Failed jobs stay in the queue; the row's ⋯ menu has Retry with the original settings pre-filled.

Exports tab

Pick an output format, name the version, hit Queue export. Each export shows up as one row. The row holds one or more downloadable files — if you exported the same data in two formats, you'll see one row with two files inside, not two separate rows. Click a row to download, re-run, or add another format.

FormatFor
LeRobotFull re-export in the lerobot dataset format. This is what WBCD submissions use.
Annotation JSONLJust the labels. Fast. Good for sharing annotations with teammates.
SnapshotVerbatim copy of the folder. No edits applied.
RecipeA saved chain of edits — re-applies the same transforms when you export.
The editor
08

Editing — first edit, then reference

Where you'll spend most of your time. The editor is for labeling episodes — keyframes, sub-goal segments, trim windows, 2D bounding boxes. Every save creates a new version of the episode; your raw recording is never modified.

The four annotation types

Before any panels: the four kinds of label you can add. Most contestants will use sub-goal segments heavily, keyframes occasionally, the others rarely.

TypeOne-line defWhen to use
Keyframe One moment. A pose snapshot at frame N. Marking the exact frame the gripper closes or releases.
Tag [start..end] A frame range with a label. Drawn as a coloured span on the timeline. Marking a phase of the trial.
Sub-goal segment A tag plus a language instruction and a review status. The main thing you'll do for WBCD. One per sub-goal phase per episode.
2D bounding box A labeled rectangle drawn on a camera frame. Tracks across frames. Marking objects (the cube, the bin) for downstream training.

Your first edit (5 minutes)

  1. Open an episode. From My Data → Files, double-click any row.
  2. Scrub. The wide black area is the viewport. Click anywhere on the timeline at the bottom to jump to that moment. Space plays / pauses; J and L step one frame at a time.
  3. Add a sub-goal segment. Press R to pick the tag tool. Click the timeline where the phase starts, then again where it ends. A coloured span appears.
  4. Fill in the label. The right panel jumps to Sub-goal segments. Type the sub-goal name and a language instruction.
  5. Mark its review status. draft while you're working, reviewed when you're done.
  6. Save. Top right → Save edit · v…. You've shipped a version. Next episode: N.

Five tools

ToolKeyUse
SelectVThe default. Pick and drag annotations.
KeyframeKClick the timeline to drop a diamond at that frame. Shift+K adds at the current position from any tool.
TagRClick-drag to draw a frame range. Becomes a sub-goal segment.
TrimTDrag the in/out handles on the timeline to crop the episode for export.
LocateLType a frame or timecode to jump there.

Versions

Click Save edit and the platform stores a new version with your name and the time. Each version remembers what changed since the last save. Open the version dropdown (top-right of the editor) to roll back to any earlier save. Your raw recording is never modified.

Two kinds of things appear in the version dropdown:

  • Versions of this episode you've saved.
  • Workspace annotation sets — re-usable labeling patterns. If you labeled the same five sub-goals on 20 pick-and-place episodes, save them as an annotation set and apply it to new episodes in one click.

Panel reference

Entity tree (left)

1 Filter input at the top is live.
2 Lists everything in the recording — robot body, cameras, joint angles, action commands.
3 Click any group to collapse it.
4 Click a camera entity to add it as a tile in the viewports.

Viewport toolbar (top)

1
Five tools on the left — Select V · Keyframe K · Tag R · Trim T · Locate L.
2
Live time + frame readout at the playhead.
3
Version dropdown — pick or load any earlier save.
4
Save edit — white dot means unsaved changes.

Viewport grid (centre)

The 3D simulator tile shows the robot in motion (or a stylised preview for unsupported embodiments). Add or remove camera tiles by clicking image entities in the tree. Bounding boxes draw directly on camera frames. Up to six cameras + the 3D tile.

Sub-goal segments tab (right)

Default right-panel tab. Card for each sub-goal segment with its label, sub-goal name, and review status. Click a card to edit it below — segment name, sub-goal name, language instruction, free-form note, status. + Segment creates a new one at the current time.

Selection tab (right)

The Selection tab shows the editable properties of whatever you clicked. Picking a keyframe shows its label, position, and a snapshot of the robot pose (gripper position, end-effector x/y/z, joint angles). Picking a tag shows its label, colour, start/end, and language instruction. Picking a bounding box shows its position and visibility range.

Timeline (bottom)

Track-label column on the left, the actual tracks on the right. The vertical line marking your current position is draggable — drag it to scrub through time. Tracks top to bottom:

  • Ruler — tick marks at each second.
  • Regions — frame-range tags drawn as coloured spans. Click to select.
  • Keyframes — diamond markers with their labels. Click to select.
  • Channels — small line graphs showing how a value changes over time (joint angle, action command, etc). Rename in the gutter, delete with the trash icon on hover, add a new one with + Track.

Export your edits

Two paths:

  • Just this clip — Export clip on the timeline transport. Ships one LeRobot bundle for this episode.
  • Whole folder — back in My Data → Exports tab (see §07). For WBCD submissions you'll usually batch-export the whole folder.

Either way, your trim window applies automatically and closed camera tiles are dropped.

Concurrent edits

Two people can't edit the same episode at once. The second editor sees a "someone else is editing this" notice. Different episodes in the same dataset are fine in parallel.

People & admin
09

Activity stream

History tab (inside My Data) = events in this project. Activity (sidebar) = events across all your projects, plus a workspace leaderboard ranking projects by submission score during a competition.

WBCD scoring

The leaderboard shows current submission scores during an active competition window. For exact scoring criteria, see the WBCD competition page (linked from the workspace banner during competitions).

11

Settings

Bottom-left of the sidebar. Two sections:

  • Profile — display name, email, default project to land on after sign-in.
  • API tokens — create / revoke personal access tokens for the CLI or scripts. Tokens inherit your account's roles.
  • Notifications — per-category toggles for email + in-app: edit completes, export ready, team invite, episode flag, import fail, weekly digest.

If you're an org admin you'll also see Organization, Quota & billing, and Audit log — covered in the Admin Guide (separate doc).

13

FAQ

If I close a project tab, do I lose anything?

No. The project moves to Recent projects ⌄ at the right of the tab strip. Click to reopen.

Can two people work on the same project at once?

Yes. You'll see each other's actions in near-real-time in the History tab. The one exception is the editor — two people can't edit the same episode at the same time.

What happens to data I remove from my project's folder?

It only leaves your project. The source dataset still lives in the catalog and any other project that pulled it in still has it. You can re-add it any time.

Why don't I see the project my teammate told me about?

They haven't invited you yet. Ask them to add you under Project members.

Can I drag-and-drop files into folders?

Not yet. Use Move to… on selected rows, or the Add to project popover from the Explorer.

My import got stuck — what happened?

If a stage takes longer than its timeout, the row will switch to failed with an error message. Click the row's ⋯ menu and pick Retry; the wizard re-opens with your original settings.

Where do I find my API tokens?

Bottom-left of the sidebar → Settings → API tokens. Tokens inherit your account's roles, so a token created by an editor can only do editor things.

What format do I submit to WBCD?

LeRobot. From your project's folder, open the Exports tab, pick the LeRobot format tile, queue the export, and upload the file when it's ready.

How do I share my annotations with a teammate?

Save them as an annotation set. From the editor, save your edits once you're happy, then in the version dropdown choose Save as annotation set. Teammates with editor access can load it on any matching episode from the same dropdown.

Getting help

  1. This guide. Click ? in the bottom-left of the sidebar to reopen.
  2. The command palette. ⌘K searches every dataset, episode, folder, project, member, and help topic. Often the fastest way.
  3. Report an issue. The Help panel's Report an issue link captures your current state and routes it to the support team.
WBCD support

During an active WBCD competition window, the #wbcd-support Slack channel has < 30 minute response time, 24/7. Outside that window, expect same-business-day on email.
